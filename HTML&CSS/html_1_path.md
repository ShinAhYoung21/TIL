# TIS: 다른 폴더에 있는 파일 링크하기: 상대경로

## Issue
* [책쾌 프로젝트](https://github.com/ShinAhYoung21/Chaek-quae) 진행 중, **메인페이지와 개별 카테고리 페이지간 링크가 제대로 작동하지 않는 문제**가 발생했다.
----

## Analysis
* ~~폴더 명이 한글이라서~~
    * 처음에는 폴더명이 한글이라서 오류가 난 줄 알았다. 하지만, 폴더명을 고치고 나서도 똑같이 오류가 발생했고, 다른 원인을 찾았다.
* 경로 설정을 잘못 해서
    * 메인 페이지는 `Chaek-quae`폴더 바로 아래, 생산성 페이지는 `Chaek-quae`폴더 하위에 있는 `productivity`폴더 내에 파일이 존재했다.
    * 그런데, 두 페이지를 연결하는 주소를 `한 폴더 내에 있는 파일 간 링크`하는 주소로 작성해서 오류가 발생한 것이었다.
----

## Solved
* 앞으로의 작업에 효율성을 더하기 위해, 절대경로가 아닌 상대경로 작성방식을 채택했다.
    * 절대경로로 작성할 경우, 관련 파일과 폴더를 한꺼번에 다른 곳으로 이동하더라도 경로를 재작성해야 하는 번거로움이 있다.
    * 하지만, 상대경로로 작성하면, 관련 파일+폴더를 모두 이동시킬 경우, 경로 수정이 필요치 않다.
* 상대경로 접목
    * 메인페이지 to 생산성 카테고리
    ``html
    <a href="./productivity/prod_1.html">생산성</a>
    ``
    * 생산성 카테고리 to 메인페이지
    ``html
    <a href="../main.html">Chack-quae</a>
    ``
----

## Learned
1. **상대경로 & 절대경로의 개념 복습**
    * 상대경로: **현재 위치**를 기준으로 위치를 표현하는 것. 예시) *우리 집의 윗집*
    * 절대경로: **현 위치와 관계 없이** 표현하는 것. 예시) *ㅁㅁ아파트 ㅇㅇ동 ###호*
2. **상대경로 표현 syntax**
    오늘 활용한 것만 정리해둔다.
    * `./` : 현재 디렉토리. 현 위치의 폴더를 의미한다.
    위의 해결 예시에서 *메인페이지 to 생산성 카테고리*의 경우,
    `./productivity/prod_1.html`에서 `./`은 메인페이지가 위치한 디렉토리. 즉, `Chaek-quae`폴더를 의미한다.
    * `../` : 현재 위치보다 상위의 디렉토리. 현 위치보다 한 단계 위의 폴더.
    *생산성 카테고리 to 메인페이지*에서,
    `../main.html`을 썼는데, 여기에서 `../`은 생산성 페이지가 위치한 `productivity`폴더보다 상위 폴더인 `Chaek-quae`를 가리키는 것이다.
